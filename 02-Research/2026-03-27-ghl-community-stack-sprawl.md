# GHL Agency Owner — Stack Sprawl & AI Coding Migration

## Signal

Post from a GHL agency owner describing their 1-year journey from no-code to AI-coded Next.js. Key data points:

### Tool Stack (8+ tools to ship client websites)
- **Site builders**: GHL → Framer → Lovable → Cursor/Antigravity + OpenCode CLI
- **Framework**: Exclusively Next.js, hosted on Vercel
- **Automation**: Self-hosted n8n (just to create GHL contacts + add tags)
- **Models used**: Opus 4.5, Sonnet 4.5, Haiku 4.5, Gemini 3 Flash/Pro, GLM-4.7, Kimi 2.5
- **Design**: Higgsfield, Freepik, Midjourney (hasn't touched Figma for web)
- **Calendars**: Still uses GHL embed

### Pain Points Expressed
1. Stack sprawl — too many tools stitched together
2. Model-hopping — different models for different tasks, no unified layer
3. Cost management — Opus "unbelievably expensive", seeks cost-to-value balance
4. Each project starts from scratch — no compounding knowledge system
5. Wants GHL to offer Vercel-style hosting + AI agents natively (unmet demand)

### Why This Matters for [[Codify]]
- This person IS our [[audience]] — GHL agency owner who codes with AI
- They've already migrated past no-code — they need the intelligence layer, not another builder
- Their n8n usage (just for GHL contact creation) is exactly what `/funnel` replaces
- Their model-hopping validates engine-agnostic positioning
- Their stack sprawl is the exact problem Codify's vault architecture solves
